## 🔍 跳跃高度问题诊断

### 可能的原因

1. **重力设置问题**
   - KCC Motor 可能有自己的重力设置
   - 检查 `Motor.Gravity` 是否被正确设置

2. **地面检测问题**
   - 角色可能没有被正确识别为"在地面上"
   - 检查 `Motor.GroundingStatus.IsStableOnGround`

3. **碰撞体设置问题**
   - Capsule Collider 高度或半径设置不当
   - 检查碰撞体是否与地面正确接触

4. **跳跃速度被覆盖**
   - 某些逻辑可能在跳跃后立即修改了速度
   - 检查是否有其他代码修改 `currentVelocity.y`

### 诊断步骤

1. **添加调试日志**
   ```csharp
   // 在跳跃时添加
   Debug.Log($"Jump! Height: {jumpHeight}, Gravity: {gravity}, Speed: {jumpSpeed}");
   Debug.Log($"IsGrounded: {Motor.GroundingStatus.IsStableOnGround}");
   ```

2. **检查 Motor 设置**
   - 在 Inspector 中查看 `KinematicCharacterMotor` 组件
   - 确认 `MaxStableSlopeAngle = 45°`
   - 确认 `MaxStepHeight = 0.3m`

3. **检查碰撞层**
   - 确保角色和地面在正确的碰撞层
   - 检查 Physics 设置中的层碰撞矩阵

## 🎯 二段跳实现建议

如果要实现二段跳，建议的设计：

### 配置参数
```csharp
[Header("Air Jump Settings")]
public bool EnableAirJump = false;
public int MaxAirJumps = 1;  // 最多空中跳跃次数
public float AirJumpHeightMultiplier = 0.8f;  // 空中跳跃高度倍数
```

### 实现要点
1. 添加 `_airJumpCount` 计数器
2. 在地面时重置计数器
3. 空中跳跃时检查计数器
4. 空中跳跃高度可以递减
