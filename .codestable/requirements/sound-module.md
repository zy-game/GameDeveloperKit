# 运行时声音播放管理

## 用户故事

- 作为玩法开发者，我希望一行调用就能播放背景音乐，而不是在场景里手动摆放和切换 AudioSource。
- 作为交互开发者，我希望按钮、技能、开门等短音效可以重复播放，并在播放结束后自动回收。
- 作为设置界面开发者，我希望能分别调整背景音和音效音量，而不是只有一个全局音量。
- 作为音频调校者，我希望游戏能切换混响、低通或快照效果，用来表达室内外、暂停、战斗等状态。

## 为什么需要

声音播放通常散落在 UI、角色、场景和状态机里；没有统一能力时，背景音乐切换、音效并发、音量设置、暂停恢复和效果切换都会各写一套，最后容易出现音量不一致、资源不释放、场景切换后音乐丢失或重复播放。

## 怎么解决

提供一个运行时声音管理能力：业务按用途播放背景音和音效，系统统一维护音轨、音量、播放中的声音实例和声音效果状态。用户可以单独控制不同音轨，也可以在游戏状态变化时切换音频效果。

## 边界

- 它不负责制作音频素材、剪辑、压缩参数或导入设置。
- 它不替代资源系统；声音素材仍需要先能被项目资源体系加载。
- 它不做音乐节拍同步、程序化作曲、语音聊天、麦克风录制或字幕系统。
- 它不做复杂音频中间件能力，比如 Wwise / FMOD 的事件图、bank 管理和实时参数曲线。
- 首版不负责把音量设置持久化到本地配置；设置界面可在外部读取后调用模块接口恢复。
