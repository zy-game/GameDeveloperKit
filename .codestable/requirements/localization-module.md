# 运行时本地化文本管理

## 用户故事

- 作为 UI 或玩法开发者，我希望用稳定的文本 key 显示界面文案，而不是把中文、英文直接写进代码和 prefab。
- 作为需要做多语言版本的人，我希望运行时可以切换语言并刷新显示，而不是重启游戏或手动重建界面。
- 作为文案维护者，我希望缺少翻译时能看到明确回退和缺失记录，而不是界面出现空文本。
- 作为框架维护者，我希望文本加载、格式化、语言切换和清理有统一规则，而不是每个模块各自写一套字典。

## 为什么需要

游戏进入多语言阶段后，文案会散在 UI、剧情、提示、调试信息和玩法反馈里。没有统一本地化能力时，硬编码文本会越来越多，改语言、查漏翻译和排查显示问题都会变成手工活，也容易出现某个界面忘记更新语言的情况。

## 怎么解决

提供一个运行时本地化能力：业务用文本 key 请求当前语言文本，系统负责加载语言内容、切换当前语言、在当前语言缺失时回退到默认语言，并把语言变化通知给需要刷新的界面或系统。缺失文本不会静默消失，而是保留可观察结果，方便后续补齐。

## 边界

- 它不负责翻译文本内容，也不提供机器翻译。
- 它不负责制作字体、字库、RTL 排版、图集或本地化图片。
- 它不替代剧情编辑器、配置编辑器或资源发布流程；这些工具只负责提供可被加载的文本数据。
- 它不自动扫描和改写所有 UI 文本；首版由业务或 UI 层显式订阅语言变化并刷新。
- 它默认处理运行时显示文本，不解决商店文案审核、平台语言包或外部运营后台流程。
